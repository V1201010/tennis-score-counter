# Tennis Score Counter

PWA-счётчик очков для большого тенниса (одиночная/парная игра), удобный для использования с телефона судьёй "с вышки".

## Стек

Статический PWA без сборки и бэкенда:
- `index.html` — вся логика приложения (HTML/CSS/JS в одном файле)
- `manifest.json` — манифест PWA
- `icon.svg` — иконка приложения
- `sw.js` — service worker (кэширование, offline)

## Архитектура

- Хранение состояния матча в `localStorage`:
  - `tennisScoreState` — текущий счёт, история ходов (для отмены)
  - `tennisSetup` — настройки матча (режим, имена игроков, формат)
- Логика счёта: очки (0/15/30/40/деюс/решающий), геймы, сеты, тай-брейк при 6:6, победа в матче по числу выигранных сетов (`setsToWin`)
- Физическая позиция на экране (`pA`/`pB`) отделена от "счётной" стороны (0/1) через `physicalToScoring()` — это позволяет менять стороны игроков (переходы по правилам теннис, либо вручную кнопкой ⇄) без изменения логики счёта
- Адаптивная раскладка: портрет — игроки сверху/снизу, ландшафт — слева/справа (вид судьи с вышки)
- Внизу слева — табло как на трансляциях (имена, подача, счёт по сетам/геймам/очкам)
- Pull-to-refresh (как на iPhone): свайп вниз сверху экрана перезагружает страницу; если это происходит на экране настройки — очищает сохранённый матч и остаётся на настройке

## Как вносить изменения

1. Изменить файлы локально (через Claude Code)
2. **Важно**: при любом изменении `index.html`, `manifest.json` или `icon.svg` увеличить `CACHE_NAME` в `sw.js` (например `tennis-score-v11`), иначе пользователи увидят старую версию из кэша service worker
3. Запустить в терминале:
```bash
cd /Users/vladislavten/tennis-score-counter
git add -A
git commit -m "описание изменений"
git push
```
4. Через 1-2 минуты изменения на сайте (может потребоваться повторная загрузка из-за обновления service worker)
